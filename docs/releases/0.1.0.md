# Tugboat 0.1.0 Release Notes

## Summary

Tugboat 0.1.0 is the first proposal-only MVP release. It ships the local CLI loop for indexing instruction files, ingesting trace bundles, running governed `llmff` audit/propose/eval pipelines, producing review artifacts, and keeping auto-apply disabled by default.

## User-Visible Changes

- Added `tugboat doctor`, `index`, `audit`, `propose`, `eval`, `report`, `optimize`, `status`, `harness check`, MCP contract tools, daemon queue commands, and operations commands.
- Added Markdown instruction discovery for `CODEX.md`, `AGENTS.md`, `SKILL.md`, runbooks, eval definitions, and policy docs.
- Added schema-validated artifacts for audit, candidate, eval, decision, report, optimization summary, operations, and release manifest outputs.
- Added review-gated apply and rollback workflows. These are not the default release posture.

## Policy Changes

- Default posture is `proposal_only`.
- `auto_apply` remains disabled by default.
- Provider-backed `llmff` runs require explicit repository policy.
- Live provider smoke checks are opt-in and cannot be enabled by environment variables alone.
- Deterministic gates reject protected policy weakening, unsafe sections, stale base hashes, overlapping edits, malformed Markdown, and restricted instruction changes.

## Storage Changes

- Tugboat writes local state under `.sidecar`.
- SQLite audit events are append-only and hash-chained.
- Run artifacts are stored under `.sidecar/runs/<run-id>/`.
- Operations evidence is stored under `.sidecar/ops` and `.sidecar/ci`.

## Verification

- `PYTHONPATH=src python -m tugboat doctor` reported `proposal_only` and `auto_apply: disabled`.
- `PYTHONPATH=src python -m tugboat index --repo . --check` completed.
- `PYTHONPATH=src python -m tugboat harness check --repo .` completed.
- `PYTHONPATH=src pytest --cov=src --cov-report=term-missing -q` passed with 674 tests and 90.77% coverage.
- `python -m build --wheel` built `dist/tugboat-0.1.0-py3-none-any.whl`.

## Rollback Notes

Rollback means returning users to the prior package and preserving `.sidecar` readability:

```bash
python -m pip install tugboat==<previous-version>
tugboat status --repo .
```

If `.sidecar` data must be restored, use `docs/ops/sidecar-backup-restore.md`.
