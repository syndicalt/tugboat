# Tugboat 1.0.0 Release Notes

## Summary

Tugboat 1.0.0 is the stable release for local-first, governed agent instruction maintenance. It builds from the 0.1.0 foundation into a documented sidecar teams can use to monitor and improve `CODEX.md`, `AGENTS.md`, `SKILL.md`, runbooks, policies, and eval definitions.

The release posture remains proposal-only by default. For 1.0.0, auto-apply remains opt-in and limited to narrow Class A lanes with policy, eval, VCS, burn-in, and rollback gates.

## User-Visible Changes Since 0.1.0

- Expanded review intelligence with rejected-edit memory, richer decision inspection, candidate comparison, highest-impact summaries, risk explanations, and rollback readiness.
- Hardened proposal provenance so audit, drift, optimizer, proposal, candidate, and reflection evidence refs remain bound to declared audit evidence.
- Expanded daemon operations with local queue recovery, read-only kill switch handling, status visibility, stale lease reporting, local socket constraints, and worktree run profiles.
- Expanded MCP read surfaces while preserving read-first behavior and avoiding direct apply authority.
- Added auto-apply preflight and shadow evidence, lane-level monitoring, pause controls, incident watch, and rollback-aware approval bundles.
- Added token-efficiency and skill-rewrite evaluation surfaces to harness and eval reporting.
- Added v1 migration, compatibility, contribution, issue-template, and security documentation.

## Compatibility And Migration

Read `docs/migration-v1.md` before upgrading a 0.x repository to 1.0.0. Read `docs/compatibility-policy.md` before depending on CLI or artifact contracts.

The supported upgrade posture is:

- back up `.sidecar`;
- run `tugboat ops migrate --repo .` before `tugboat ops migrate --repo . --apply`;
- block older binaries from silently reading newer `.sidecar/version.json` schemas;
- keep provider-backed `llmff` paths explicitly policy-gated.

## Verification Checklist

Retain release evidence for the exact release commit:

```bash
tugboat doctor --repo .
tugboat index --repo . --check
tugboat harness check --repo .
tugboat ci --repo .
python -m pytest --cov=src --cov-report=term-missing -q
python -m build --wheel
python -m twine check dist/<wheel>.whl
```

Installed-wheel smoke must install the built wheel in a clean virtual environment and run at least `tugboat doctor`, `tugboat index --repo . --check`, and `tugboat harness check --repo .`.

`tugboat ops release-manifest --repo .` must retain coverage, build, twine, install-smoke, harness, CI, and security-review evidence.

## Known Limitations

- No public network daemon.
- No hosted web UI.
- No multi-user team server.
- No MCP path for direct apply.
- No provider credential management through MCP.
- No broad auto-apply beyond narrow Class A policy lanes.
- Provider-backed `llmff` execution requires explicit `llmff.allow_network: true` policy and reviewed provider configuration.
