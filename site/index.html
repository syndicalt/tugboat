Agent instructions that can be observed, improved, and rolled back.

Tugboat monitors the operating layer around coding agents: CODEX.md, AGENTS.md, SKILL.md, runbooks, policies, and eval definitions. It turns real traces into bounded proposals with provenance, validation, and human review.

Mission

Make the harness layer legible enough to improve safely.

Coding agents already depend on local instructions, runbooks, policies, and evals. Those files quietly shape behavior, but they rarely have telemetry, review trails, or rollback discipline.

Tugboat treats that layer as production infrastructure. It reads traces, identifies drift, proposes narrow edits, validates impact, and leaves an auditable trail before anything is applied.

Workflow

A proposal loop with evidence at every handoff.

Tugboat keeps model orchestration separate from authority. llmff runs bounded manifests; Tugboat owns policy, artifacts, evals, VCS checks, and review state.

Decision trace

Every proposal carries its own review packet.

Evidence
Trace refs, instruction snapshots, candidate rationale, and eval output.
Authority
Policy gates, VCS state, protected writes, and read-only kill switch checks.
Recovery

Automation

Capabilities

Production controls for changing the text that controls agents.

01

Instruction observability

Indexes instruction files, runbooks, policies, evals, and trace evidence so drift has concrete refs.

02

Bounded proposals

Generates narrow diffs with rationale, expected behavior, risk classification, and review artifacts.

03

Eval-gated optimization

Runs acceptance, held-out, token-efficiency, and skill behavior checks before recommending change.

04

VCS apply and rollback

Applies through proposal, branch, commit, or PR modes with provenance and rollback plans.

05

Safe automation lane

Class A auto-apply stays opt-in, shadowed, policy-bound, and blocked by active incidents.

06

Local operator control

Runs locally with credential-free fixture paths, read-only MCP surfaces, and auditable sidecar state.
